## 本节解决什么问题

C 语言中，用 `malloc/free` 管理动态内存；C++ 中，用 `new/delete`。这些手动管理有严重问题：

1. **忘记 `delete`** → 内存泄漏
2. **重复 `delete`** → 未定义行为（通常崩溃）
3. **提前 `return` 或抛异常** → `delete` 永远不会执行
4. **共享所有权困难** → 谁负责释放？什么时候释放？

智能指针利用 RAII 自动管理内存：你不用写 `delete`，资源在不需要时自动释放。

## 这个特性是什么

智能指针是 RAII 理念的具体实现——把动态分配的资源（`new` 出来的东西）交给一个栈上的对象管理，这个对象的析构函数会自动 `delete`。

STL 提供了三种智能指针：

| 智能指针 | 所有权 | 说明 |
|:---|:---|:---|
| `std::unique_ptr` | 独占 | 只有一个所有者，不可复制，可移动。**默认第一选择**。 |
| `std::shared_ptr` | 共享 | 多个所有者，引用计数为 0 时释放。有额外开销。 |
| `std::weak_ptr` | 弱引用 | 观察 `shared_ptr` 管理的对象，不增加引用计数，解决循环引用。 |

## C++ 标准版本

- `std::unique_ptr`：C++11（替代了废弃的 `std::auto_ptr`）
- `std::shared_ptr` / `std::weak_ptr`：C++11
- `std::make_unique`：C++14
- `std::make_shared`：C++11

## 需要的头文件

```cpp
#include <memory>
```

## 基本语法

```cpp
// unique_ptr：独占
auto p1 = std::make_unique<Type>(构造参数...);
auto p2 = std::move(p1);  // p1 转移给 p2，p1 变为 nullptr

// shared_ptr：共享
auto sp1 = std::make_shared<Type>(构造参数...);
auto sp2 = sp1;  // sp1 和 sp2 共享同一对象，引用计数 +1

// weak_ptr：弱引用
std::weak_ptr<Type> wp = sp1;
auto sp3 = wp.lock();  // 尝试获取 shared_ptr（对象可能已释放）
```

## 对比 new/delete vs 智能指针

| 操作 | new/delete | 智能指针 |
|:---|:---|:---|
| 分配 | `Type* p = new Type(...);` | `auto p = std::make_unique<Type>(...);` |
| 使用 | `p->method();` | `p->method();` （和使用指针一样） |
| 释放 | `delete p;` | 不需要！自动释放 |
| 异常安全 | ❌ 手动保证 | ✅ 保证 |
| 所有权清晰 | ❌ 靠注释约定 | ✅ 类型系统强制 |

## 示例代码

### 示例 1：unique_ptr 基本用法（对比 new/delete）

```cpp
#include <iostream>
#include <memory>

struct Person
{
    std::string name;
    int age;

    Person(const std::string& n, int a) : name(n), age(a)
    {
        std::cout << "Person " << name << " created\n";
    }
    ~Person()
    {
        std::cout << "Person " << name << " destroyed\n";
    }
};

int main()
{
    // ❌ 旧写法：手动 new/delete
    Person* rp = new Person("Alice", 25);
    rp->name = "Alice_new";
    delete rp;  // 必须手动释放！
    std::cout << "---\n";

    // ✅ unique_ptr：自动管理
    {
        auto up = std::make_unique<Person>("Bob", 30);
        std::cout << up->name << " is " << up->age << "\n";
        // 离开作用域，up 自动 delete
    }
    std::cout << "---\n";

    // 再创建一个看看自动销毁
    auto up2 = std::make_unique<Person>("Charlie", 35);
    std::cout << up2->name << " will be auto-destroyed at end of main\n";

    return 0;
}
```

**运行结果**：

```
Person Alice created
Person Alice destroyed
---
Person Bob created
Bob is 30
Person Bob destroyed
---
Person Charlie created
Charlie will be auto-destroyed at end of main
Person Charlie destroyed
```

### 示例 2：在示例 1 基础上，unique_ptr 的所有权转移

```cpp
#include <iostream>
#include <memory>

struct Student
{
    std::string name;
    Student(const std::string& n) : name(n)
    {
        std::cout << "Student " << name << " created\n";
    }
    ~Student()
    {
        std::cout << "Student " << name << " destroyed\n";
    }
};

// 函数接收 unique_ptr 的所有权
void take_ownership(std::unique_ptr<Student> s)
{
    std::cout << "Function now owns: " << s->name << "\n";
    // s 离开作用域，Student 被销毁
}

int main()
{
    auto s1 = std::make_unique<Student>("Alice");

    // s1 不能拷贝
    // auto s2 = s1;  // ❌ 编译错误！unique_ptr 不可拷贝

    // s1 可以移动
    auto s2 = std::move(s1);  // s1 转移给 s2，s1 变为 nullptr
    std::cout << "s1 is null? " << (s1 == nullptr ? "yes" : "no") << "\n";

    // 传递给函数（移动所有权）
    take_ownership(std::move(s2));
    std::cout << "s2 is null? " << (s2 == nullptr ? "yes" : "no") << "\n";

    return 0;
}
```

**运行结果**：

```
Student Alice created
s1 is null? yes
Function now owns: Alice
Student Alice destroyed
s2 is null? yes
```

### 示例 3：在示例 2 基础上，shared_ptr 共享所有权

```cpp
#include <iostream>
#include <memory>

struct Counter
{
    int value;
    Counter(int v) : value(v)
    {
        std::cout << "Counter(" << value << ") created\n";
    }
    ~Counter()
    {
        std::cout << "Counter(" << value << ") destroyed\n";
    }
};

int main()
{
    std::shared_ptr<Counter> sp1;

    {
        auto sp2 = std::make_shared<Counter>(100);

        sp1 = sp2;  // sp1 和 sp2 共享，引用计数 = 2
        std::cout << "ref count (in scope): " << sp1.use_count() << "\n";
        std::cout << "value = " << sp1->value << "\n";
    }
    // sp2 离开作用域，但 sp1 还在，引用计数 = 1，Counter 没有被销毁！

    std::cout << "ref count (out of scope): " << sp1.use_count() << "\n";
    std::cout << "value = " << sp1->value << "\n";

    // sp1 离开 main 作用域，引用计数 = 0，Counter 被销毁

    return 0;
}
```

**运行结果**：

```
Counter(100) created
ref count (in scope): 2
value = 100
ref count (out of scope): 1
value = 100
Counter(100) destroyed
```

### 示例 4：在示例 3 基础上，weak_ptr 解决循环引用

```cpp
#include <iostream>
#include <memory>

struct Node
{
    std::string name;
    std::weak_ptr<Node> other;  // 用 weak_ptr 避免循环引用！

    Node(const std::string& n) : name(n)
    {
        std::cout << "Node " << name << " created\n";
    }
    ~Node()
    {
        std::cout << "Node " << name << " destroyed\n";
    }
};

int main()
{
    auto a = std::make_shared<Node>("A");
    auto b = std::make_shared<Node>("B");

    // 互相引用（但用 weak_ptr，不增加引用计数）
    a->other = b;
    b->other = a;

    std::cout << "a use_count: " << a.use_count() << "\n";
    std::cout << "b use_count: " << b.use_count() << "\n";

    // 通过 weak_ptr.lock() 访问
    if (auto ptr = a->other.lock())
    {
        std::cout << "A's other is: " << ptr->name << "\n";
    }

    return 0;
    // a 和 b 离开作用域，引用计数为 0，正确释放！没有循环引用问题
}
```

**运行结果**：

```
Node A created
Node B created
a use_count: 1
b use_count: 1
A's other is: B
Node B destroyed
Node A destroyed
```

## 示例中的关键语法解释

| 示例 | 讲了什么 | 新出现的语法 | 为什么这样写 | 注意事项 |
|:---|:---|:---|:---|:---|
| 示例 1 | unique_ptr vs new/delete | `std::make_unique<T>()`、自动析构 | 用 make_unique 创建，不用手动 delete | make_unique 在 C++14 才加入 |
| 示例 2 | unique_ptr 所有权转移 | `std::move()`、nullptr 检查 | unique_ptr 不可拷贝，通过 move 转移所有权 | move 后原指针变为 nullptr |
| 示例 3 | shared_ptr 共享所有权 | `std::make_shared<T>()`、`use_count()` | 引用计数为 0 时自动 delete | shared_ptr 有引用计数开销 |
| 示例 4 | weak_ptr 解决循环引用 | `std::weak_ptr`、`lock()` | weak_ptr 不增加引用计数，lock() 返回 shared_ptr | 循环引用中如果一个用 shared_ptr 会导致永不释放 |

## 常见错误

**错误 1：把 new 出来的指针直接赋给另一个 unique_ptr 而不是转移**

```cpp
auto p1 = std::make_unique<int>(42);
auto p2 = p1;  // ❌ 编译错误！unique_ptr 不可拷贝
```

正确做法：`auto p2 = std::move(p1);`

**错误 2：shared_ptr 循环引用**

```cpp
struct A {
    std::shared_ptr<B> b;  // 如果 B 也有 shared_ptr<A> a，则循环引用！
};
```

正确做法：其中一方用 `std::weak_ptr` 打破循环。

**错误 3：用 shared_ptr 的原始指针创建另一个 shared_ptr**

```cpp
auto sp = std::make_shared<int>(42);
int* rp = sp.get();
auto sp2 = std::shared_ptr<int>(rp);  // ❌ sp2 和 sp 有各自的控制块！会重复 delete！
```

正确做法：从已有 shared_ptr 拷贝构造：`auto sp2 = sp;`

**错误 4：忘记 `make_unique` / `make_shared`**

```cpp
std::unique_ptr<int> p(new int(42));  // 能用但不是最佳
```

正确做法：`auto p = std::make_unique<int>(42);`（更安全、更高效）。

## 使用建议

1. **默认用 `unique_ptr`**：绝大多数场景不需要共享所有权。
2. **需要共享所有权时才用 `shared_ptr`**：有引用计数开销，不要滥用。
3. **用 `make_unique` / `make_shared` 而不是 `new`**：异常安全，代码简洁。
4. **用 `weak_ptr` 打破循环引用**：如果两个对象互相引用，其中一方用 weak_ptr。
5. **不要用智能指针管理栈上的变量**：`int x = 5; unique_ptr<int> p(&x);` ❌

## 小结

- 智能指针是 RAII 的典范：自动管理动态内存，不需要手动 `delete`。
- `unique_ptr`：独占所有权，默认选择，不可拷贝但可移动。
- `shared_ptr`：共享所有权，引用计数为 0 时释放，有开销。
- `weak_ptr`：弱引用，不增加引用计数，解决循环引用，通过 `lock()` 访问。
- 永远用 `make_unique` / `make_shared` 创建智能指针。
