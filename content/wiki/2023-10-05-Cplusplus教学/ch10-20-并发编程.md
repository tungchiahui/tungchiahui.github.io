## 本节解决什么问题

现代计算机有多核 CPU，一个程序只用一个线程等于浪费了其他核的计算能力。另外，有些任务天生适合"同时做"——UI 响应 + 后台下载、同时处理多个连接、并行计算。

C++11 引入了标准化的多线程支持，让你不需要依赖平台特定的 API（如 pthread、Win32 线程），一套代码跨平台运行。

## 这个特性是什么

C++ 标准库提供了：

- `std::thread`：创建和管理线程。
- `std::mutex` + `std::lock_guard` / `std::unique_lock`：互斥量，保护共享数据。
- `std::condition_variable`：线程间条件同步。
- `std::atomic`：原子操作，无锁的线程安全变量。

## C++ 标准版本

C++11（C++14/17/20 持续增加如 `shared_mutex`、`jthread`、`latch`、`barrier` 等）。

## 需要的头文件

```cpp
#include <thread>              // std::thread, std::this_thread
#include <mutex>               // std::mutex, std::lock_guard, std::unique_lock
#include <condition_variable>  // std::condition_variable
#include <atomic>              // std::atomic
```

## 基本语法

```cpp
// 创建线程
std::thread t(func, args...);
t.join();        // 等待线程结束
t.detach();      // 分离线程（后台运行）

// 互斥量
std::mutex mtx;
std::lock_guard<std::mutex> lock(mtx);      // 自动加锁/解锁（RAII）
std::unique_lock<std::mutex> ulock(mtx);     // 更灵活的锁

// 原子变量
std::atomic<int> counter{0};
++counter;  // 线程安全的自增
```

## 锁的对比

| 锁类型 | 加锁 | 解锁 | 可手动解锁 | 可转移 | 推荐度 |
|:---|:---|:---|:---|:---|:---|
| 手动 `lock()`/`unlock()` | 手动 | 手动 | ✅ | — | ❌ 不推荐 |
| `std::lock_guard` | 自动（构造） | 自动（析构） | ❌ | ❌ | ✅ 简单场景首选 |
| `std::unique_lock` | 自动（构造） | 自动（析构） | ✅ | ✅ | ✅ 需要灵活控制时用 |

## 示例代码

### 示例 1：创建两个线程并发执行

```cpp
#include <iostream>
#include <thread>
#include <string>

void task1(const std::string& msg)
{
    for (int i = 0; i < 3; ++i)
    {
        std::cout << "Task1: " << msg << " [" << i << "]\n";
    }
}

void task2(int n)
{
    for (int i = 0; i < 3; ++i)
    {
        std::cout << "Task2: " << n * i << " [" << i << "]\n";
    }
}

int main()
{
    // 创建两个线程
    std::thread t1(task1, "hello");
    std::thread t2(task2, 10);

    // 等待两个线程完成
    t1.join();
    t2.join();

    std::cout << "Both threads finished\n";

    return 0;
}
```

**运行结果**（输出顺序可能不同）：

```
Task1: hello [0]
Task2: 0 [0]
Task1: hello [1]
Task2: 10 [1]
Task1: hello [2]
Task2: 20 [2]
Both threads finished
```

### 示例 2：在示例 1 基础上，数据竞争问题和 mutex 解决

```cpp
#include <iostream>
#include <thread>
#include <mutex>

int counter = 0;            // 共享变量
std::mutex mtx;             // 保护 counter 的锁

void increment(int times)
{
    for (int i = 0; i < times; ++i)
    {
        // lock_guard：构造时加锁，析构时解锁（RAII）
        std::lock_guard<std::mutex> lock(mtx);
        ++counter;
    }
}

int main()
{
    std::thread t1(increment, 100000);
    std::thread t2(increment, 100000);

    t1.join();
    t2.join();

    std::cout << "counter = " << counter << " (expected 200000)\n";

    return 0;
}
```

**运行结果**：

```
counter = 200000 (expected 200000)
```

> **思考**：如果把 `lock_guard` 那行删掉再运行几次，结果可能不是 200000——这就是数据竞争。

### 示例 3：在示例 2 基础上，unique_lock 更灵活的加锁

```cpp
#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx;
int shared = 0;

void flexible_task()
{
    // unique_lock：可以延迟加锁
    std::unique_lock<std::mutex> lock(mtx, std::defer_lock);

    // 做不需要锁的事情...
    std::cout << "Doing preparation...\n";

    // 需要时才加锁
    lock.lock();
    shared += 10;
    std::cout << "shared = " << shared << " (in critical section)\n";
    lock.unlock();  // 可以提前解锁

    // 做不需要锁的事情...
    std::cout << "Doing cleanup...\n";

    // lock 离开作用域时如果还持有锁，自动解锁
}

int main()
{
    std::thread t1(flexible_task);
    std::thread t2(flexible_task);

    t1.join();
    t2.join();

    std::cout << "final shared = " << shared << "\n";

    return 0;
}
```

**运行结果**（输出顺序可能不同）：

```
Doing preparation...
shared = 10 (in critical section)
Doing cleanup...
Doing preparation...
shared = 20 (in critical section)
Doing cleanup...
final shared = 20
```

### 示例 4：在示例 3 基础上，用 std::atomic 实现无锁计数

```cpp
#include <iostream>
#include <thread>
#include <atomic>
#include <vector>

std::atomic<int> atomic_counter{0};

void atomic_increment(int times)
{
    for (int i = 0; i < times; ++i)
    {
        ++atomic_counter;  // 原子操作，不需要 mutex！
    }
}

int main()
{
    std::vector<std::thread> threads;
    for (int i = 0; i < 4; ++i)
    {
        threads.emplace_back(atomic_increment, 100000);
    }

    for (auto& t : threads)
    {
        t.join();
    }

    std::cout << "atomic_counter = " << atomic_counter
              << " (expected 400000)\n";

    return 0;
}
```

**运行结果**：

```
atomic_counter = 400000 (expected 400000)
```

### 示例 5：条件变量——生产者消费者模式

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>

std::queue<int> data_queue;
std::mutex mtx;
std::condition_variable cv;
bool done = false;

void producer()
{
    for (int i = 1; i <= 5; ++i)
    {
        {
            std::lock_guard<std::mutex> lock(mtx);
            data_queue.push(i);
            std::cout << "Produced: " << i << "\n";
        }
        cv.notify_one();  // 通知消费者
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }

    // 生产完毕
    {
        std::lock_guard<std::mutex> lock(mtx);
        done = true;
    }
    cv.notify_one();
}

void consumer()
{
    while (true)
    {
        std::unique_lock<std::mutex> lock(mtx);
        cv.wait(lock, [] { return !data_queue.empty() || done; });

        while (!data_queue.empty())
        {
            int val = data_queue.front();
            data_queue.pop();
            lock.unlock();
            std::cout << "Consumed: " << val << "\n";
            lock.lock();
        }

        if (done && data_queue.empty())
        {
            break;
        }
    }
}

int main()
{
    std::thread p(producer);
    std::thread c(consumer);

    p.join();
    c.join();

    return 0;
}
```

**运行结果**：

```
Produced: 1
Consumed: 1
Produced: 2
Consumed: 2
Produced: 3
Consumed: 3
Produced: 4
Consumed: 4
Produced: 5
Consumed: 5
```

## 示例中的关键语法解释

| 示例 | 讲了什么 | 新出现的语法 | 为什么这样写 | 注意事项 |
|:---|:---|:---|:---|:---|
| 示例 1 | 创建和 join 线程 | `std::thread`、`join()` | 创建即启动，join 等待完成 | join 前检查 `joinable()` |
| 示例 2 | 数据竞争和 mutex | `std::mutex`、`std::lock_guard` | lock_guard 是 RAII，自动管理锁 | 锁的范围尽量小 |
| 示例 3 | unique_lock 灵活控制 | `std::unique_lock`、`defer_lock` | 延迟加锁、手动解锁等场景 | 开销比 lock_guard 稍大 |
| 示例 4 | 原子操作 | `std::atomic<int>`、`++` | 无锁线程安全，比 mutex 快 | 仅适用于简单操作 |
| 示例 5 | 生产者消费者 | `std::condition_variable`、`wait()/notify_one()` | cv.wait 配合 lambda 判断条件 | wait 需要用 unique_lock |

## 常见错误

**错误 1：忘记 join 或 detach，线程析构时程序崩溃**

```cpp
{
    std::thread t(func);
}  // ❌ t 离开作用域时还是 joinable，程序 std::terminate！
```

正确做法：在 `t` 离开作用域前调用 `t.join()` 或 `t.detach()`。

**错误 2：线程函数访问已销毁的局部变量**

```cpp
void start_thread()
{
    int local = 42;
    std::thread t([&local] { std::cout << local; });  // ❌ local 可能已销毁！
    t.detach();
}
```

正确做法：按值捕获或用智能指针确保生命周期。

**错误 3：在锁的范围内做耗时操作**

```cpp
{
    std::lock_guard<std::mutex> lock(mtx);
    std::this_thread::sleep_for(10s);  // ❌ 锁持有 10 秒！
}
```

正确做法：只在访问共享数据时持有锁，耗时操作移到锁外面。

**错误 4：向线程传递引用时忘记 `std::ref`**

```cpp
int x = 10;
std::thread t(func, x);  // x 被值拷贝！
```

正确做法：`std::thread t(func, std::ref(x));` 传递引用。

## 使用建议

1. **优先用 `std::lock_guard`**：简单、安全、RAII，覆盖 80% 场景。
2. **只对简单类型用 `std::atomic`**：`++`、`--`、`+=` 等操作是原子的，更复杂的操作用 mutex。
3. **锁的范围越小越好**：只锁住访问共享数据的代码块。
4. **`join()` 是等待线程完成**：注意顺序，避免死锁。
5. **多读少写的场景用 `std::shared_mutex`（C++17）**：允许并发读，写时独占。
6. **C++20 引入了 `std::jthread`**：析构时自动 join，比 `std::thread` 更安全。

## 小结

- `std::thread` 创建和管理线程，`join()` 等待完成，`detach()` 后台运行。
- `std::mutex` + `std::lock_guard` 保护共享数据，RAII 自动管理锁。
- `std::unique_lock` 更灵活（延迟加锁、手动解锁、配合条件变量）。
- `std::atomic<T>` 无锁原子操作，适合简单计数器。
- `std::condition_variable` 实现线程间通知/等待。
- 数据竞争是多线程的常见问题，用 mutex 或 atomic 解决。

### 工程拓展

在 ROS2 中，多线程回调处理（`rclcpp::CallbackGroup`）、互斥量保护节点间共享数据等，都直接使用了 C++ 的并发工具。在 FreeRTOS（嵌入式）中，虽然 API 不同，但互斥量、信号量、队列等概念与 C++ 标准库是对应的。学好 C++ 并发编程，对这些库的理解会非常快。
